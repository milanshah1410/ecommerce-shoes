🔴 Critical

Raw SQL errors exposed to users — When creating a coupon without required fields, the full SQL query, database name (ecommerce_shoes), host (127.0.0.1), port (3306), and table schema are displayed in an alert dialog. This is a critical security vulnerability.
Orders API returns HTTP 500 — /api/admin/orders intermittently returns 500 errors, causing the Orders page to show "No records found" even when orders exist. The dashboard showed 1 order (# ORD-SWFCLOVNIX, ₹14,157.64 pending) which never appeared in the Orders list.
Add Product broken — boolean fields not sent — When creating a product, the "Product Tags" checkboxes (Featured, Trending, New Arrival, Best Seller) don't submit boolean values correctly. The backend rejects the request with "The is featured field must be true or false. (and 3 more errors)", making it impossible to create a new product.

🟠 High

Movement Log not recording stock changes — After successfully adding 10 units via Stock In, the Movement Log shows "No movements recorded." Stock is updated but the audit trail is broken.
Profile page loads with empty fields — When navigating to the Profile page, all fields (First Name, Last Name, Email, Mobile) are empty. The current user's data is not populated.
Logout triggered by scrolling — During testing, scrolling on the Brands page triggered an unexpected logout. This points to an event propagation bug.
Unintended navigation from modal actions — Clicking Cancel/Confirm buttons on modals sometimes navigates to a completely different section (e.g., confirming category deletion navigated to Brands, cancelling product edit navigated to Categories).

🟡 Medium

Global search bar non-functional — Typing in the top search bar and pressing Enter yields no results and no navigation.
